Lesson 5. Git.

Основы

Git — это набор консольных утилит, которые отслеживают и фиксируют изменения в файлах (чаще всего речь идет об исходном коде программ, но вы можете использовать его для любых файлов на ваш вкус). С его помощью вы можете откатиться на более старую версию вашего проекта, сравнивать, анализировать, сливать изменения и многое другое. Этот процесс называется контролем версий. Существуют различные системы для контроля версий. Вы, возможно, о них слышали: SVN, Mercurial, Perforce, CVS, Bitkeeper и другие.

Про Git в интернете огромное количество информации. Если вы в поиске сдалаете запрос "что такое Git" Вам выдаст пребольшой список информации (от Википедии до рукописных статей) из которого Вы можете выбрать на свой вкус. Вот примеры некоторых статей, которые могут помочь вам разобраться:

  • Введение - Что такое Git?
  • Git и GitHub: что это такое и в чём разница

Использование

  • git clone https://github.com/s..... - Клонировать репозиторий на локальный компьютер
  • git pull - Получить все изменения из репозитория
  • git checkout -b branchname - Создать новую ветку и переключится на нее
  • git add . - Добавить изменения, которые сделали локально
  • git commit -m "commit text" - Подписать изменения
  • git push origin branch_name - Закачать последний коммит на сервер
